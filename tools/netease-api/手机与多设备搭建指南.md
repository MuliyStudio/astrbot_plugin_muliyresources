# 网易云解析后端搭建指南（手机 / 多设备版）

> 适用：你在**手机**上也装了 AstrBot，想让插件的「网易云语音名片」功能在手机端也能用。
> 配套一键脚本见同目录 `setup_termux.sh`（**Termux 原生 或 Termux + proot Ubuntu 都能跑，自动识别**）。
>
> ⚠️ 先记住一句话：**`setup_termux.sh` 本身就是「在手机上部署 netease-api」**。
> 它不是「部署完之后再额外装点什么」——跑这个脚本 = 手机已经有 netease-api 了，之后只去插件里填地址即可。

---

## 0. 先搞懂一件事：为什么要自建

插件 `core/netease.py` 里，网易云功能**只支持自建后端**（`wyy_custom_url`），
公共解析站（wyapi / qzxdp）早在 v1.9.3 就移除了（被网易云 WAF 拦截）。

所以规则很简单：**每台跑这个插件的设备，都要能连到一个 NeteaseCloudMusicApi 实例。**

✅ **最省事的做法**：搭**一个**实例（手机本机 或 一台常开服务器 / 云），
让手机和电脑的插件填**同一个地址**，而不是每台设备各搭一个。

---

## 1. 方案一览（按「手机怎么跑 AstrBot」选）

| 你的 AstrBot 跑在哪 | 推荐方案 | wyy_custom_url 填什么 |
|---------------------|----------|------------------------|
| **手机 Termux / Ubuntu(proot) 里** | 方案 A：本机跑脚本 | `http://127.0.0.1:3000` |
| **手机连远程服务器** | 方案 B/C：服务器上建，手机填服务器地址 | `http://<服务器IP>:3000` |
| **多设备都要用** | 方案 B/C：云端/服务器建一个，共用 | 公网或内网 IP |

> ⚠️ **千万别搞混 127.0.0.1 的语义**：
> - AStrBot 和 NeteaseCloudMusicApi **在同一个环境**（同一台手机 Termux）→ `127.0.0.1:3000` ✅ 正确
> - AStrBot 跑在 **Docker 容器里**，API 在宿主机/另一容器 → `127.0.0.1` ❌ 错（那是容器自己），要填宿主机真实 IP
> （上面这条是 `README.md` 里 Docker 场景的坑，手机 Termux 本机场景不踩。）

---

## 2. 方案 A：手机本机跑（最简单，零服务器）

适合：AstrBot 装在手机 Termux 或 Termux + proot Ubuntu 里，不需要别的设备访问。

> 你的环境是 **AstrBot 装在 Ubuntu（proot）里、不在 Docker**。这种场景下：
> - `127.0.0.1` **完全可用**（Ubuntu 与 netease-api 同处一个环境，不像 Docker 要换宿主机 IP）。
> - 一键脚本 `setup_termux.sh` 已自动识别 Ubuntu，会用 `apt` 装 Node.js（不再是 Termux 的 `pkg`）。

### 方法一：一键脚本（推荐）
把同目录 `setup_termux.sh` 传到手机，在** Ubuntu 终端**里执行：

```bash
bash setup_termux.sh
```

脚本会自动：装 Node.js（apt）→ 后台启动 API（端口 3000）→ 生成 `start_api.sh` 便于重启。
看到 `server running @ http://localhost:3000` 即成功。

然后在 AStrBot 后台 → 插件配置 → `muliyresources`：
```
wyy_custom_url = http://127.0.0.1:3000
```

### 方法二：手动两行（Ubuntu 内）
```bash
apt-get update && apt-get install -y nodejs npm
npx NeteaseCloudMusicApi@latest
```

### ⚠️ 手机本机的致命坑：后台被杀
Termux 切到后台，系统几秒~几分钟就会杀掉进程，API 一停插件就解析失败。解决三选一：
1. 把 Termux 加入系统「电池优化白名单 / 省电例外」
2. 在** Termux 原生终端**（不是 Ubuntu 里）执行 `termux-wake-lock`，并保持终端在前台
3. 改用方案 B/C 把服务放常开机器（最稳，彻底避坑）

---

## 3. 方案 B：云端一键（多设备共用，最推荐）

搭一次，手机 + 电脑 + 任意设备填同一个地址，且常开不掉线。

### Vercel（真·几步点击）
1. 打开 https://github.com/Binaryify/NeteaseCloudMusicApi → **Fork** 到你账号
2. 打开 https://vercel.com → New Project → Import 你刚 Fork 的仓库
3. Framework Preset 选 **Other** → 直接 Deploy
4. 拿到 `https://xxx.vercel.app`，填进 `wyy_custom_url`

> ⚠️ **两个针对本插件的注意点**：
> - Vercel 版部分接口需要额外 `realIP` 参数（`/song/url?id=xxx&realIP=1.2.3.4`），
>   而插件代码调用时**没带这个参数**，可能导致解析失败。建议 Vercel **绑定国内备案域名**后再用，或直接选方案 C。
> - Vercel 在国内访问偏慢。

### 小 VPS / 云服务器（更稳，无 realIP 烦恼）
在服务器上（任选）：
```bash
# 官方镜像一行
docker run -d -p 3000:3000 --name netease_api binaryify/netease_cloud_music_api
```
或复用本插件自带的 compose（把 `tools/netease-api/` 整目录传到服务器）：
```bash
cd tools/netease-api && docker compose up -d --build
```
然后所有设备填：
```
wyy_custom_url = http://<服务器公网或内网IP>:3000
```

## 4. 配置到插件（每台设备都填）

AstrBot 后台 → 插件配置 → `muliyresources`：

| 配置项 | 填什么 | 说明 |
|--------|--------|------|
| `wyy_custom_url` | 实例地址（见上表） | **必填**，否则功能不工作 |
| `wyy_cookie` | 黑胶会员 `MUSIC_U` Cookie（可选） | 填了才能解析 VIP/付费歌曲，否则这类歌返回 `null` |
| `wyy_music_type` | `standard` / `exhigh` / `lossless`（可选） | 音质，VIP 音质需配合 `wyy_cookie` |

---

## 5. 自检（确认通了再放心用）

在能访问到实例的机器上跑插件自带脚本：
```bash
python tools/check_netease_api.py --url http://<实例地址>:3000
```
返回 `✅ 直链可用` 即齐活。

也可直接 curl 验证：
```bash
curl "http://<实例地址>:3000/song/url?id=28921655&level=standard"
```

## 6. 常见坑速查

- **解析失败 / 连不上** → 99% 是 `wyy_custom_url` 填错。Docker 容器里别填 `127.0.0.1`；跨机填真实 IP。
- **返回 200 但 url 为 null** → 该曲是 VIP/版权限制，换普通歌；或配置 `wyy_cookie`。
- **手机本机跑了但过会儿失效** → Termux 被后台杀了，看方案 A 的保活三招。
- **ffmpeg 缺失** → 插件退化为发完整音频（仍可用）；要截 60 秒高潮需在 AstrBot 环境装 ffmpeg。

## 7. Termux 里 AstrBot 占着终端、打不了命令怎么办

**现象**：AstrBot 在前台终端运行，终端被它占用，你输不进 shell 命令、也跑不了 `setup_termux.sh`。
**原因**：AstrBot 进程占着前台，按键都被它接收了。**千万别按 Ctrl+C**（会直接杀掉 AstrBot）。

### 解法一：开第二个 Termux 会话（最简单，推荐）
- Termux 中从**左边缘向右滑** → 「New session / 新建会话」
- 新会话进入 Ubuntu（若 AstrBot 装在 Ubuntu）：`proot-distro login ubuntu`
- 在新会话里跑脚本：`bash /path/to/setup_termux.sh`
- 第一个会话的 AstrBot 继续跑，互不影响

### 解法二：用 AstrBot 自带 Web 管理面板（最优雅，可完全不碰终端）
- AstrBot 自带 Web 面板，端口在 `astrbot_config.yaml` 的 `dashboard` 配置项（常见 8080）
- 手机浏览器打开 `http://localhost:<端口>` → 在面板里直接改插件配置 `wyy_custom_url`
- 部署完 netease-api 后，填地址用面板最省事

### 解法三：tmux 把 AstrBot 后台化（常驻首选）
```bash
tmux new -s ab          # 进入 tmux，在里面启动 AstrBot
# ... 启动 astrbot ...
Ctrl+b d                # detach 出来，AstrBot 在后台继续跑
tmux attach -t ab       # 随时回来看日志
```
之后想输命令，直接开新 Termux 会话或新 tmux 窗口即可，互不干扰。

> 想让 AstrBot 开机自启/彻底后台，可配合 `nohup astrbot &` 或系统服务（如 Termux:Boot 插件）。
```
